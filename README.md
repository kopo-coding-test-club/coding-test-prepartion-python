# 🚀 Coding Test Study (Python)

### 이 저장소는 코딩 테스트 스터디의 문제 풀이 기록과 코드 리뷰를 위한 공간입니다.

### 매주 정해준 문제를 풀고, 서로의 코드 공유하는 방식으로 스터디를 진행합니다.

***

# 📅 운영방식

### 1. 묹제 풀이 : 매주 정해진 분량의 문제를 해결합니다.

### 2. 코드 업로드 : 각자 본인의 이름으로 브랜치를 만드신 후 수요일 오후 4시까지 main 브랜치로 PULL REQUEST(PR)을 보냅니다

### 3. 코드 리뷰 : 수요일 모임 당일, main 브랜치에서 취합된 코드를 기준으로 코드 리뷰 진행합니다.

***

# 📁 폴더 구조 가이드

### CODING-TEST-CLUB/
### └── [Month]/                # 예: April
###    └── [Week]/              # 예: First_Week
###        └── [Name]/          # 예: KimBuYeon
###            └── [File].py    # 예: 12947_Lv1_HarshadNumber.py

***

# ✍️ 파일 네이밍 규칙

### 1. 파일명은 프로그래머스 고유 번호와 난이도를 포함하여 작성합니다.

### 2. 문제번호 확인: 프로그래머스 문제 URL 맨 뒤의 숫자 (예: .../lessons/12947)

### 3. 파일명 에시 : 12947_Lv1_HarshadNumber.py

### 4. 주의 : 혹시나 발생할 수 있는 인코딩 문제로 인한 한글 꺠짐 문제를 방지하기 위해 파일명은 영어로 작성합니다. 문제 출제하시는 분이 한글명과 영문명 두 버전으로 문제를 올려주세요

***
# 💬 커밋 메시지 규칙

### 일관된 커밋 기록을 위해 아래의 말머리(Prefix)를 사용해 주세요.

| 태그 | 의미 | 예시 |
| :--- | :--- | :--- |
| ✨ **DONE** | 문제를 완전히 풀었을 때 | `DONE: 12947_HarshadNumber` |
| 🏗️ **UNDONE** | 문제를 완전히 풀었을 때 | `UNDONE: 12947_HarshadNumber` |
| ⚙️ **Fix** | 코드 수정 및 리팩토링 | `Fix: 12947_HarshadNumber 효율성 개선` |
| 📝 **Docs** | README 등 문서 수정 | `Docs: README 파일 업로드` |
| 🚚 **Chore** | 폴더 구조 변경 등 기타 작업 | `Chore: 폴더 구조 변경` |
